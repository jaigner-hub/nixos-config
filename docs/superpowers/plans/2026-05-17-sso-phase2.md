# SSO Implementation Plan (Phase 2)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire Immich and Paperless on the homelab to the existing Pocket-ID IdP at `auth.tail1ec6c3.ts.net`, using each app's native OIDC support and matching the Phase 1 secret-distribution pattern.

**Architecture:** Per app: create a Pocket-ID client (admin-group-restricted) in the admin UI, drop a `/etc/<app>-oidc.{env,json}` file on the host carrying client_id + client_secret, then wire the NixOS module to consume that via its native `secretsFile` (Immich) or `environmentFile` (Paperless) option. Keep regular password login enabled during rollout — only flip to OIDC-only after a clean week.

**Tech Stack:** NixOS unstable, `services.immich` (native OAuth), `services.paperless` (django-allauth + the `openid_connect` provider), Pocket-ID v2.x.

**Spec:** `docs/superpowers/specs/2026-05-17-sso-phase2-design.md`

**Phase 3 (Jellyfin) is a separate, later plan.**

---

## Notes for the implementer

- Run Nix commands from `/home/enum/Projects/nixos-config`.
- Builds: `nixos-rebuild build --flake .#<host>` produces a `./result` symlink without touching any live machine.
- Validate fleet-wide: `nix flake check`.
- Deploys: `scripts/deploy.sh <host>` probes reachability then runs `colmena apply --on <host>`.
- Commit style: lowercase, short, host-prefixed (`immich: ...`, `paperless: ...`). No Conventional Commits.
- Hosts reached over the tailnet (`<host>.tail1ec6c3.ts.net`); SSH as `jeff`.
- All OIDC client secrets are secrets — **never** commit any of them. They live in files outside the Nix store, referenced through systemd `EnvironmentFile=` / the module's native `secretsFile` option.
- Pocket-ID client-group restriction has a session-cache gotcha: after toggling a user's group memberships in Pocket-ID, the user must **log out of Pocket-ID itself** and back in for the new group set to take effect (see `project_pocket_id_bootstrap.md`).
- Tailnet certs renew weekly via the existing `tailscale-cert` units on both hosts — no first-deploy intervention needed; both hosts already have a working cert.


Verify:

```bash

```

Expected:

```
-rw------- 1 immich immich ... /etc/immich-oidc.json
```

- [ ] **Step 2: Edit `machines/immich/configuration.nix`**

Find the existing `services.immich = { ... };` block (currently `machines/immich/configuration.nix:47-52`). Replace it with:

```nix
  services.immich = {
    enable = true;
    host = "127.0.0.1";
    port = 2283;
    mediaLocation = dataDir;

    # OAuth/OIDC via Pocket-ID at auth.tail1ec6c3.ts.net. clientId and
    # clientSecret are placeholders here — they're overlaid at service
    # start from /etc/immich-oidc.json via secretsFile, which keeps the
    # actual values out of the Nix store. See the Phase 2 SSO spec.
    settings.oauth = {
      enabled = true;
      issuerUrl = "https://auth.tail1ec6c3.ts.net";
      clientId = "set-by-secrets-file";
      clientSecret = "set-by-secrets-file";
      scope = "openid email profile";
      buttonText = "Login with Pocket-ID";
      autoRegister = true;
      autoLaunch = false;

      # Immich's Android/iOS app uses the app.immich:/// custom URL
      # scheme as its OIDC callback. Pocket-ID won't accept a non-HTTPS
      # callback (reasonable: custom-scheme deep links are spoofable).
      # mobileOverrideEnabled makes the Immich server expose
      # /api/oauth/mobile-redirect, an HTTPS endpoint registered as a
      # callback in Pocket-ID, which then JS-redirects to the deep link.
      mobileOverrideEnabled = true;
      mobileRedirectUri =
        "https://immich.youtalklikeafag.com/api/oauth/mobile-redirect";
    };

    # JSON overlay merged into settings at service start. Only contains
    # the oauth.clientId and oauth.clientSecret fields. Provisioned
    # out-of-band — see /etc/immich-oidc.json (mode 0600 immich:immich).
    secretsFile = "/etc/immich-oidc.json";
  };
```

(The rest of the file — `imports`, NFS mount, nginx, cloudflared, restic, etc. — stays untouched.)

## Task 3: End-to-end Immich OIDC flow + commit (MANUAL)

Verify the full browser + mobile flow lands you in the existing `jeff` Immich library (not a fresh empty account). Reconcile if it forks.

**Files:**
- Modify: `machines/immich/configuration.nix` (commit only)

- [ ] **Step 1: Browser flow**

In the incognito window from Task 2 step 6, click **Login with Pocket-ID**. Browser flow:

1. Redirect to `https://auth.tail1ec6c3.ts.net/authorize?...`
2. Pocket-ID auto-logs you in (existing tailnet session) or prompts for passkey.
3. Pocket-ID prompts for consent on `openid email profile`. Approve.
4. Redirect back to `https://immich.youtalklikeafag.com/auth/login?code=...`
5. Immich exchanges the code, lands you in the dashboard.

Verify in Immich:

- Top-right user menu shows your Pocket-ID email/username.
- The library shows your existing photos (not an empty grid). If empty, you've forked into a new account — see step 3 below.

- [ ] **Step 2: Mobile flow**

On the phone, open the Immich app, log out of the current session, choose **Login with OAuth** (or equivalent — UI label has shifted across versions). Tap through:

1. App opens the system browser to `https://auth.tail1ec6c3.ts.net/authorize?...`
2. Pocket-ID prompts for passkey (or uses an existing session from the system browser).
3. Browser redirects to `https://immich.youtalklikeafag.com/api/oauth/mobile-redirect?code=...`
4. The server's mobile-redirect endpoint JS-redirects to `app.immich:///oauth-callback?code=...`
5. Phone OS hands the deep link back to the Immich app, which exchanges the code and lands you in the library.

Verify: the mobile app shows the existing photo library, top-right user menu reflects the Pocket-ID identity.

If step 4 fails (browser stays on the redirect page), the `mobileRedirectUri` value in the Nix config doesn't match the Pocket-ID callback list — re-check both contain exactly `https://immich.youtalklikeafag.com/api/oauth/mobile-redirect`.

- [ ] **Step 3: Reconcile if a duplicate account was created**

If browser-flow step 1 landed you in an empty library (fresh OIDC user instead of attaching to existing `jeff`):

```bash
ssh jeff@immich.tail1ec6c3.ts.net 'sudo -u postgres psql immich -c "SELECT id, email, name FROM users ORDER BY \"createdAt\";"'
```

Expected: two rows — the original `jeff` user and a new OIDC-derived user with the same email.

Fix by attaching the OIDC subject to the original user via the Immich admin UI:

1. Log in as the OIDC-derived account (the fresh one) — it's now the admin since `autoRegister` makes the first OIDC user admin.
2. Go to **Administration → Users**.
3. Promote the original `jeff` account to admin.
4. Log out, log back in as `jeff` via password (regular login still enabled).
5. Go to **Administration → Users → [new OIDC user] → Delete**.
6. In the original `jeff` user's profile (admin → users → edit), set the **OAuth ID** field to the OIDC `sub` from the deleted user's record.
7. Log out, log back in via Pocket-ID — should now land in the original library.

If the Immich UI doesn't expose an OAuth ID field directly (varies across versions), the underlying column is `users.oauthId` in Postgres — set it manually:

```bash
ssh jeff@immich.tail1ec6c3.ts.net 'sudo -u postgres psql immich'
# In psql:
#   UPDATE users SET "oauthId" = '<OIDC_SUB>' WHERE email = 'jeff.aigner@gmail.com';
#   \q
```

The OIDC `sub` for your Pocket-ID account is visible at `https://auth.tail1ec6c3.ts.net/account` (under "User ID" or similar) — or in Pocket-ID admin → Users → click your user.

- [ ] **Step 2: Edit `machines/paperless/configuration.nix`**

Find the existing `services.paperless = { ... };` block (currently `machines/paperless/configuration.nix:27-46`). Replace it with:

```nix
  services.paperless = {
    enable = true;
    address = "127.0.0.1";
    port = 28981;

    # Initial admin user. Provision the password file out-of-band before
    # first start (paperless reads it on first boot to seed the user):
    #   echo -n 'mypass' | sudo install -m 600 -o paperless -g paperless \
    #     /dev/stdin /etc/paperless-admin-pass
    passwordFile = "/etc/paperless-admin-pass";

    settings = {
      PAPERLESS_URL = "https://${publicFqdn}";
      PAPERLESS_OCR_LANGUAGE = "eng";
      PAPERLESS_TIME_ZONE = "America/Chicago";
      # Trust the reverse proxy; without this, CSRF rejects uploads.
      # Both nginx (tailnet) and cloudflared (public) connect from loopback.
      PAPERLESS_TRUSTED_PROXIES = "127.0.0.1";

      # OIDC via Pocket-ID. PAPERLESS_APPS *appends* to Django's
      # INSTALLED_APPS — safe to set alongside paperless's own apps. The
      # actual provider config (with the client_secret) lives in
      # PAPERLESS_SOCIALACCOUNT_PROVIDERS, loaded from environmentFile
      # below so the secret stays out of the Nix store.
      PAPERLESS_APPS = "allauth.socialaccount.providers.openid_connect";

      # First OIDC login auto-creates the local account (allauth
      # otherwise stops on a "complete signup" form).
      PAPERLESS_SOCIAL_AUTO_SIGNUP = "True";

      # Match OIDC users to existing local users by email so the first
      # OIDC login attaches to the existing `admin` account instead of
      # forking a new one. EMAIL_VERIFICATION=none skips allauth's
      # confirmation email step (we don't have outbound SMTP).
      PAPERLESS_ACCOUNT_EMAIL_VERIFICATION = "none";
      PAPERLESS_ACCOUNT_AUTHENTICATION_METHOD = "username_email";
    };

    # Carries PAPERLESS_SOCIALACCOUNT_PROVIDERS (entire JSON blob —
    # contains the client_secret). Mode 0600 paperless:paperless on the
    # host. Single line: paperless reads as KEY=VALUE.
    environmentFile = "/etc/paperless-oidc.env";
  };
```

(The rest of the file — `imports`, nginx, cloudflared, tailscale-cert, restic — stays untouched.)
